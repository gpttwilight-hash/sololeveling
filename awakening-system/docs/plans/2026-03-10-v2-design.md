## 1. Контекст и цели

**Ситуация:** MVP (T-001..T-036) реализован. Базовая геймификация работает.
**Проблема:** Продукт технически готов, но не готов к публичному запуску — нет достаточного WOW-эффекта, AI-дифференциатора и маркетинговой основы.
**Цель v2.0:** Превратить рабочий прототип в продукт, за который платят.

**Метрики успеха:**
- 500+ пользователей в первый месяц
- 5% конверсия Free → Pro
- 30-дневный retention >40%
- Product Hunt: Top 5 дня запуска

## 2. Приоритизированные блоки работы

### Блок A — Cinematic Onboarding (Высший приоритет)

**Почему первым:** Первое впечатление необратимо. Сейчас онбординг — стандартная форма. Нужна последовательность, которая продаёт идею до первого квеста.

**Поток:**
1. `/onboarding/welcome` — тёмный экран, анимированный текст: *"Система обнаружила нового Охотника"*
2. `/onboarding/goals` — выбор 3 целей из карточек (Здоровье / Карьера / Обучение / Финансы / Отношения / Творчество)
3. `/onboarding/attributes` — объяснение 5 атрибутов, привязка к выбранным целям
4. `/onboarding/first-quest` — AI автоматически создаёт первые 3 квеста под цели
5. `/onboarding/rank` — *"Ранг E присвоен. Путь начат."* → redirect на dashboard

**Технически:** Framer Motion для переходов, localStorage для промежуточного состояния, Server Action для сохранения целей и создания первых квестов.

---

### Блок B — AI Quest Advisor (Главный Pro-дифференциатор)

**Почему:** Это единственная фича, которую Habitica, Streaks и другие конкуренты не имеют. Даёт прямое обоснование цены Pro.

**Функционал:**
- Страница `/goals` — пользователь вводит долгосрочную цель + дедлайн
- AI (Claude API через Supabase Edge Function) разбивает цель на квесты по неделям
- Каждое воскресенье в 20:00 → push-уведомление → Weekly Review: краткий AI-анализ прогресса
- Monthly Report — PDF/страница с инсайтами за месяц

**Ограничения по тирам:**
- Free: 1 цель, ручные квесты
- Pro: неограниченные цели, AI-генерация, Weekly Review, Monthly Report

**Архитектура:**
```
Client → POST /api/ai/generate-quests
       → Supabase Edge Function
       → Anthropic Claude API
       → INSERT quests (bulk)
       → Return quest list
```

**Промпт-стратегия:** структурированный JSON output, max 7 квестов на неделю, с атрибутами STR/INT/CHA/DIS/WLT автоматически.

### Блок C — Dashboard Redesign

**Изменения:**
- Убрать card-grid → вертикальный поток с hero-квестом (самый важный квест дня — 40% экрана)
- Status Bar всегда сверху: `[E] Охотник | 🔥 14 | 💰 340`
- Radar chart атрибутов → на главный экран (свернуть/развернуть)
- Пустой стейт: не пустота, а *"Система ожидает действий, Охотник"* с CTA создать квест

**Micro-interactions (Framer Motion):**
- Завершение квеста → XP-бар анимация + монеты летят в счётчик
- Level Up → fullscreen overlay с rank badge (0.8 сек)
- Стрик → пульсирующий огонь рядом с числом
- Кнопки → spring animation при нажатии (scale 0.95)

---

### Блок D — Weekly Boss Battle

**Механика:**
- Каждый понедельник в 00:00 (cron) → генерируется Boss of the Week
- Босс = набор из 5-7 квестов под общей целью (например: "Пробудить Охотника" = 3 тренировки + 2 часа учёбы + 1 книга)
- Прогресс-бар Босса на дашборде (HP от 100 до 0 по мере выполнения)
- Победа = уникальный badge + 500 bonus XP

**Таблица боссов:** 52 заготовленных босса на год, с ротацией по сезонам.

---

### Блок E — Монетизация и Paywall

**Тиры:**
| Tier | Цена | Лимиты |
|------|------|--------|
| Free | $0 | 5 квестов/день, 1 цель, нет AI |
| Hunter Pro | $7.99/мес или $59/год | Всё без лимитов + AI + Boss + Reports |
| Lifetime | $149 (launch: $89) | Всё навсегда |

**Paywall-механика:**
- Soft paywall: при попытке создать 6-й квест → modal "Стань Hunter Pro"
- Hard paywall: AI Advisor, Weekly Boss, Monthly Report — только Pro
- Бесплатные пользователи видят заблюренный Monthly Report с CTA

**Платёжная система:** Stripe (через Supabase + Stripe webhook для управления подписками).

## 3. Технические улучшения для production-ready

- **Optimistic UI** — все действия мгновенные, синхронизация в фоне
- **PWA Push Notifications** — стрик-напоминание в 21:00 если <3 квестов выполнено
- **Offline mode** — завершать квесты без интернета (IndexedDB + sync queue)
- **Dynamic OG Images** — при шаринге достижения генерируется уникальная карточка
- **Sentry** — error tracking с первого дня продакшна
- **PostHog** — event tracking для понимания где пользователи отваливаются

---

## 4. Что НЕ делаем в v2.0 (YAGNI)

- Guild System — сложно, нужна критическая масса пользователей
- Marketplace темп — контент сначала, потом платформа
- Native app (iOS/Android) — PWA достаточно для MVP
- Собственный платёжный процессинг — только Stripe
